# Part 1: Pricing & Margin Strategies

## 1.1 Industry Margin Benchmarks

The following benchmarks synthesize data from industry reports, Damodaran datasets, and
LatAm-specific research. Margins in Latin America tend to be 2-5 percentage points lower
than US/EU equivalents due to higher input cost volatility, informal competition, and
currency risk. Where LatAm-specific figures are available they are noted; otherwise US/global
benchmarks are adjusted with a LatAm discount factor.

### Restaurants / Food Service

| Metric | Quick Service | Casual Dining | Fine Dining | Food Trucks / Street Food |
|---|---|---|---|---|
| Gross Margin | 60-70% | 55-65% | 60-70% | 65-75% |
| Net Profit Margin | 4-6% | 2-4% | 5-10% | 8-15% |
| Food Cost % | 28-35% | 30-38% | 28-35% | 25-32% |
| Labor Cost % | 25-30% | 30-35% | 30-40% | 15-25% |

**LatAm-specific notes:**
- In Mexico, fondas and cocinas economicas operate on razor-thin net margins (2-5%) but

- Brazilian rodizios and buffets per kilo achieve 5-8% net margins through high throughput.
- Street food vendors across LatAm often achieve the highest net margins (10-15%) due to
  minimal overhead, but with lower absolute revenue.
- Top-performing restaurants in the region report net margins near 10%, while the industry
  average hovers between 3-5%.

## 1.2 Pricing Strategies Playbook

### Strategy 1: Cost-Plus Pricing

**When to use:** Products with well-defined input costs (food, retail goods, manufacturing).
Best for businesses new to pricing or in highly commoditized markets.

**Formula:**

```
Selling Price = Total Cost x (1 + Markup Percentage)
```

**TypeScript implementation for Kitz AI agents:**

```typescript
interface CostPlusInput {
  directMaterialCost: number;
  directLaborCost: number;
  overheadAllocation: number;
  targetMarkupPercent: number;
}

function calculateCostPlusPrice(input: CostPlusInput): {
  totalCost: number;
  sellingPrice: number;
  grossMargin: number;
  grossMarginPercent: number;
} {
  const totalCost =
    input.directMaterialCost +
    input.directLaborCost +
    input.overheadAllocation;

  const sellingPrice = totalCost * (1 + input.targetMarkupPercent / 100);
  const grossMargin = sellingPrice - totalCost;
  const grossMarginPercent = (grossMargin / sellingPrice) * 100;

  return {
    totalCost: Math.round(totalCost * 100) / 100,
    sellingPrice: Math.round(sellingPrice * 100) / 100,
    grossMargin: Math.round(grossMargin * 100) / 100,
    grossMarginPercent: Math.round(grossMarginPercent * 10) / 10,
  };
}

// Example: A restaurant dish in Mexico City
const tacoPlatter = calculateCostPlusPrice({
  directMaterialCost: 45, // MXN - ingredients
  directLaborCost: 15,    // MXN - prep time allocation
  overheadAllocation: 10,  // MXN - rent, utilities, etc.
  targetMarkupPercent: 200, // 200% markup = ~33% food cost ratio
});
// Result: sellingPrice = 210 MXN, grossMarginPercent = 66.7%
```

### Strategy 2: Value-Based Pricing

**When to use:** Services with high perceived value, specialized expertise, or outcomes-based
work (consulting, medical, design, legal). Ideal for differentiated businesses.

**How to determine perceived value:**

1. **Customer research** -- Survey target market to gauge willingness to pay. Ask open-ended
   questions about what outcomes matter most and what they would pay for those outcomes.
2. **Segment identification** -- Different segments have different price thresholds, enabling
   differential pricing (e.g., corporate clients vs. individuals).
3. **Unique value analysis** -- Identify what extra value you deliver that competitors do not.
4. **Outcome quantification** -- For B2B services, quantify the ROI your service delivers.

```typescript
interface ValueBasedPricingInput {
  costToDeliver: number;
  competitorPrice: number;
  perceivedValueMultiplier: number; // 1.0 = parity, 1.5 = 50% more value
  customerSegment: 'budget' | 'standard' | 'premium';
}

const segmentDiscounts: Record<string, number> = {
  budget: 0.7,    // Price at 70% of perceived value
  standard: 0.85, // Price at 85% of perceived value
  premium: 1.0,   // Price at 100% of perceived value
};

function calculateValueBasedPrice(input: ValueBasedPricingInput): {
  recommendedPrice: number;
  floorPrice: number;
  ceilingPrice: number;
  valueToCustomer: number;
} {
  const valueToCustomer = input.competitorPrice * input.perceivedValueMultiplier;
  const segmentFactor = segmentDiscounts[input.customerSegment];
  const recommendedPrice = valueToCustomer * segmentFactor;
  const floorPrice = input.costToDeliver * 1.3; // Minimum 30% margin
  const ceilingPrice = valueToCustomer; // Never exceed perceived value

  return {
    recommendedPrice: Math.max(floorPrice, Math.round(recommendedPrice * 100) / 100),
    floorPrice: Math.round(floorPrice * 100) / 100,
    ceilingPrice: Math.round(ceilingPrice * 100) / 100,
    valueToCustomer: Math.round(valueToCustomer * 100) / 100,
  };
}

// Example: A marketing agency in Bogota, Colombia
const brandingPackage = calculateValueBasedPrice({
  costToDeliver: 2_000_000,        // COP - labor + tools
  competitorPrice: 5_000_000,      // COP - average competitor price
  perceivedValueMultiplier: 1.4,   // 40% more value via better portfolio
  customerSegment: 'standard',
});
// Result: recommendedPrice ~5,950,000 COP
```

### Strategy 3: Competitive Pricing

**How to research competitors in LatAm:**

1. **Mystery shopping** -- Visit competitor locations or websites; note prices for identical
   or similar services.
2. **Mercado Libre / Amazon Mexico** -- For products, search your category and sort by
   best-selling to see dominant price points.
3. **Google Maps reviews** -- Customers sometimes mention prices in reviews.
4. **WhatsApp/social media inquiry** -- In LatAm, many businesses quote prices via WhatsApp;
   send inquiries as a prospective customer.
5. **Industry associations** -- CANACO (Mexico), FENALCO (Colombia), SEBRAE (Brazil) publish
   sector pricing benchmarks.

```typescript
interface CompetitorPriceData {
  competitorName: string;
  price: number;
  qualityScore: number; // 1-10
  marketShare: 'high' | 'medium' | 'low';
}

function calculateCompetitivePrice(
  competitors: CompetitorPriceData[],
  ownQualityScore: number,
  strategy: 'penetration' | 'parity' | 'premium'
): { recommendedPrice: number; rationale: string } {
  const avgPrice =
    competitors.reduce((sum, c) => sum + c.price, 0) / competitors.length;
  const avgQuality =
    competitors.reduce((sum, c) => sum + c.qualityScore, 0) / competitors.length;

  const qualityDifferential = ownQualityScore / avgQuality;

  const strategyMultipliers = {
    penetration: 0.85,  // 15% below market
    parity: 1.0,        // Match market
    premium: 1.15,      // 15% above market
  };

  const basePrice = avgPrice * qualityDifferential;
  const recommendedPrice = basePrice * strategyMultipliers[strategy];

  const rationale =
    `Market average: ${avgPrice.toFixed(2)}. ` +
    `Quality differential: ${(qualityDifferential * 100 - 100).toFixed(1)}%. ` +
    `Strategy: ${strategy}. ` +
    `Recommended: ${recommendedPrice.toFixed(2)}.`;

  return {
    recommendedPrice: Math.round(recommendedPrice * 100) / 100,
    rationale,
  };
}
```

### Strategy 4: Psychological Pricing

**Key tactics proven effective in LatAm markets:**

1. **Charm Pricing (precios terminados en 9)** -- Prices ending in .99 or .90 exploit
   the left-digit bias. Research in Brazilian e-commerce confirms this drives impulse
   purchases, especially among middle and lower-income segments. Example: $199.90 MXN
   instead of $200 MXN.

2. **Anchoring (anclaje de precios)** -- Display original price alongside the discounted
   price. LatAm consumers in particular respond strongly to visible "antes/ahora"
   (before/now) comparisons. Brazilian e-commerce platforms frequently use this tactic.

3. **Bundle Pricing (paquetes)** -- Combine services or products at a perceived discount.
   Example: "Corte + barba + cejas = $250 MXN" vs. individual prices totaling $320 MXN.

4. **Decoy Pricing (precio senuelo)** -- Offer three tiers where the middle option provides
   the best perceived value. The expensive option makes the middle look reasonable.

5. **"Meses sin intereses" (Interest-free installments)** -- Extremely powerful in LatAm
   where credit access is limited. Even for services, offering "3 meses sin intereses"
   dramatically increases conversion.

6. **Multiple Purchase Discounts (3x2, 2x1)** -- The "3 por 2" promotion is deeply
   embedded in LatAm retail culture, commonly used to clear perishables or introduce new
   products.

```typescript
interface PsychologicalPricingOptions {
  basePrice: number;
  currency: 'MXN' | 'BRL' | 'COP' | 'ARS' | 'PEN' | 'CLP';
}

function applyCharmPricing(options: PsychologicalPricingOptions): number {
  const { basePrice, currency } = options;

  // Different charm endings by currency convention
  const charmEndings: Record<string, number> = {
    MXN: 0.90,  // Mexico: $199.90
    BRL: 0.99,  // Brazil: R$199.99
    COP: 900,   // Colombia: $199,900 COP (prices are in thousands)
    ARS: 0.99,  // Argentina: $199.99
    PEN: 0.90,  // Peru: S/199.90
    CLP: 0,     // Chile: prices are whole numbers, end in 990
  };

  if (currency === 'COP') {
    // Colombian prices operate in thousands
    return Math.floor(basePrice / 1000) * 1000 + charmEndings[currency];
  }
  if (currency === 'CLP') {
    // Chilean prices end in 990
    return Math.floor(basePrice / 1000) * 1000 + 990;
  }

  return Math.floor(basePrice) + charmEndings[currency];
}

function generateAnchoredPrice(
  currentPrice: number,
  anchorMultiplier: number = 1.4
): { anchorPrice: number; displayPrice: number; savingsPercent: number } {
  const anchorPrice = Math.round(currentPrice * anchorMultiplier);
  const savingsPercent = Math.round(
    ((anchorPrice - currentPrice) / anchorPrice) * 100
  );
  return {
    anchorPrice,
    displayPrice: currentPrice,
    savingsPercent,
  };
}

function generateBundlePrice(
  items: { name: string; individualPrice: number }[],
  discountPercent: number = 15
): {
  individualTotal: number;
  bundlePrice: number;
  savings: number;
  itemNames: string[];
} {
  const individualTotal = items.reduce((sum, i) => sum + i.individualPrice, 0);
  const bundlePrice = Math.round(
    individualTotal * (1 - discountPercent / 100)
  );
  return {
    individualTotal,
    bundlePrice,
    savings: individualTotal - bundlePrice,
    itemNames: items.map((i) => i.name),
  };
}

// Example: Barbershop in Medellin
const barberBundle = generateBundlePrice([
  { name: 'Corte clasico', individualPrice: 25_000 },
  { name: 'Perfilado de barba', individualPrice: 15_000 },
  { name: 'Diseno de cejas', individualPrice: 10_000 },
], 20);
// Result: Bundle = 40,000 COP vs. 50,000 COP individual (20% savings)
```

### Strategy 5: Dynamic Pricing

**When to use:** Tourism, hospitality, events, perishable goods, ride-sharing,
delivery services.

**Seasonal adjustments for LatAm:**

| Season/Event | Country | Price Adjustment | Duration |
|---|---|---|---|
| Carnival | Brazil | +30-50% (tourism services) | 1 week (Feb/Mar) |
| Buen Fin | Mexico | -10-30% (promotional) | 4 days (November) |
| Christmas/Navidad | All LatAm | +10-20% (gifts/food) | Dec 1-24 |
| Semana Santa | All LatAm | +20-40% (tourism) | 1 week (Mar/Apr) |
| Back-to-School | Varies | +5-15% (education/supplies) | 2-4 weeks |
| Low tourism season | Caribbean/Mexico | -15-30% | May-October |

```typescript
interface DynamicPricingInput {
  basePrice: number;
  currentDemandIndex: number;   // 0.0-2.0, where 1.0 = normal
  inventoryLevel: number;       // 0.0-1.0, where 1.0 = full
  dayOfWeek: number;            // 0=Sun, 6=Sat
  isHolidaySeason: boolean;
  competitorPriceIndex: number; // 0.5-1.5, where 1.0 = at parity
}

function calculateDynamicPrice(input: DynamicPricingInput): {
  adjustedPrice: number;
  priceChange: number;
  factors: string[];
} {
  let multiplier = 1.0;
  const factors: string[] = [];

  // Demand adjustment (max +/- 30%)
  const demandFactor = 1 + (input.currentDemandIndex - 1) * 0.3;
  multiplier *= demandFactor;
  if (demandFactor !== 1) {
    factors.push(`Demand: ${demandFactor > 1 ? '+' : ''}${((demandFactor - 1) * 100).toFixed(0)}%`);
  }

  // Inventory scarcity (low stock = higher price, max +20%)
  if (input.inventoryLevel < 0.3) {
    const scarcityFactor = 1 + (0.3 - input.inventoryLevel) * 0.67;
    multiplier *= scarcityFactor;
    factors.push(`Low stock: +${((scarcityFactor - 1) * 100).toFixed(0)}%`);
  }

  // Weekend premium (Fri-Sun, +5-10%)
  if (input.dayOfWeek >= 5 || input.dayOfWeek === 0) {
    multiplier *= 1.07;
    factors.push('Weekend: +7%');
  }

  // Holiday season
  if (input.isHolidaySeason) {
    multiplier *= 1.15;
    factors.push('Holiday season: +15%');
  }

  // Competitive response (dampen extreme deviations)
  if (input.competitorPriceIndex < 0.9) {
    multiplier *= 0.95;
    factors.push('Competitor undercut: -5%');
  }

  // Cap maximum adjustment at +/- 50%
  multiplier = Math.max(0.5, Math.min(1.5, multiplier));

  const adjustedPrice = Math.round(input.basePrice * multiplier * 100) / 100;
  const priceChange = Math.round((multiplier - 1) * 100 * 10) / 10;

  return { adjustedPrice, priceChange, factors };
}
```

## 1.3 Price Elasticity in LatAm

### Overview

Latin American consumers are among the most price-sensitive in the world due to:

- **Persistent inflation** averaging ~6% annually across the region, with extremes in
  Argentina (100%+) and Venezuela.
- **Purchasing power erosion** -- household purchasing power in LatAm fell approximately
  25% due to inflation in recent years. Consumers responded by buying smaller portions
  and switching to cheaper alternatives (private labels, generics).
- **Cash-dependent economies** -- a significant portion of transactions remain cash-based,
  limiting dynamic pricing adoption in some sectors.
- **Strong value-seeking culture** -- the "buscar ofertas" (bargain-hunting) mindset is
  deeply embedded. Consumers actively compare prices across channels.

### Elasticity by Segment

| Segment | Elasticity | Behavior | Implication |
|---|---|---|---|
| Basic necessities (food, hygiene) | Low (-0.3 to -0.6) | Must buy regardless | Small price increases absorbed |
| Discretionary (clothing, electronics) | High (-1.2 to -2.0) | Defer or substitute | Price increases cause volume loss |
| Services (beauty, dining out) | Medium (-0.7 to -1.2) | Trade down, reduce frequency | Offer value tiers |
| Premium/luxury | Low (-0.3 to -0.8) | Status-driven | Less price-sensitive segment |
| Subscription/recurring | Medium (-0.8 to -1.3) | Evaluate alternatives at renewal | Lock in annual contracts |

### Country-Specific Price Sensitivity

| Country | Sensitivity Level | Key Driver | Pricing Implication |
|---|---|---|---|
| Argentina | Extreme | Hyperinflation, currency instability | Price in USD or adjust monthly; offer smaller packages |
| Venezuela | Extreme | Economic crisis | Dollar pricing essential |
| Mexico | High | Inflation, competition from informal market | Charm pricing + installments effective |
| Brazil | High | Income inequality, inflation history | Parcelamento (installments) is table stakes |
| Colombia | Medium-High | Growing middle class but inflation concerns | Value bundles, prima seasons important |
| Chile | Medium | Most stable economy in region | Standard pricing strategies work |
| Peru | Medium-High | Informal economy competition | Competitive pricing essential |

### Strategies for Price-Sensitive Markets

1. **Smaller package sizes** -- "sachets strategy" (selling shampoo in single-use packets)
   is widespread in LatAm. Apply this principle to services: offer 30-min sessions, single
   treatments, trial classes.
2. **Installment payments ("meses sin intereses")** -- In Brazil, "parcelamento" is so
   standard that not offering it is a competitive disadvantage. In Mexico, "6 MSI" (6
   months without interest) on credit card is extremely common.
3. **Everyday Low Price (EDLP)** -- The Walmart/Bodega Aurrera model is widely imitated.
   Stable, transparent pricing builds trust.
4. **3x2 and 2x1 promotions** -- The most recognizable promotional format in LatAm retail.
   Use for clearing perishables, introducing new products, or driving trial.

```typescript
interface ElasticityAnalysis {
  currentPrice: number;
  currentVolume: number;
  estimatedElasticity: number; // negative number (e.g., -1.2)
  proposedPriceChange: number; // percentage (e.g., 10 for 10% increase)
}

function predictVolumeImpact(input: ElasticityAnalysis): {
  newPrice: number;
  expectedVolume: number;
  volumeChange: number;
  currentRevenue: number;
  projectedRevenue: number;
  revenueChange: number;
  recommendation: string;
} {
  const newPrice = input.currentPrice * (1 + input.proposedPriceChange / 100);
  const volumeChangePercent = input.estimatedElasticity * input.proposedPriceChange;
  const expectedVolume = Math.round(
    input.currentVolume * (1 + volumeChangePercent / 100)
  );

  const currentRevenue = input.currentPrice * input.currentVolume;
  const projectedRevenue = newPrice * expectedVolume;
  const revenueChange =
    ((projectedRevenue - currentRevenue) / currentRevenue) * 100;

  let recommendation = '';
  if (revenueChange > 2) {
    recommendation = 'RECOMENDAR: El aumento de precio mejorara los ingresos.';
  } else if (revenueChange > -2) {
    recommendation =
      'NEUTRAL: Impacto minimo en ingresos. Considere si el margen mejora.';
  } else {
    recommendation =
      'PRECAUCION: El aumento de precio reducira los ingresos. Considere alternativas.';
  }

  return {
    newPrice: Math.round(newPrice * 100) / 100,
    expectedVolume,
    volumeChange: Math.round(volumeChangePercent * 10) / 10,
    currentRevenue: Math.round(currentRevenue),
    projectedRevenue: Math.round(projectedRevenue),
    revenueChange: Math.round(revenueChange * 10) / 10,
    recommendation,
  };
}
```

## 1.5 When and How to Raise Prices

### Signs It Is Time to Raise Prices

1. **Your margins are shrinking** -- Costs have risen but prices have not.
2. **You are turning away business** -- Demand exceeds capacity.
3. **You have not raised prices in 12+ months** -- Inflation alone erodes margins.
4. **Competitor prices are higher** -- You may be undervaluing your offering.
5. **Customer acquisition is easy** -- Low resistance suggests you are priced too low.
6. **You have improved quality/service** -- Enhanced value justifies higher prices.
7. **Customer complaints about price are rare** -- If nobody ever objects, you are too cheap.

### Price Increase Methods

| Method | Description | Risk Level | Best For |
|---|---|---|---|
| Across-the-board | Raise all prices equally (e.g., +8%) | Medium | Inflation adjustment |
| Selective | Raise prices on best sellers, keep loss leaders | Low | Retail, food |
| Grandfathering | New customers pay more, existing keep old price | Low | Services, subscriptions |
| New packaging | Repackage/rebrand at higher price point | Low | Products, services |
| Reduce discounts | Keep list price, stop offering discounts | Very Low | Businesses over-discounting |
| Add premium tier | Keep current prices, add expensive option | Very Low | All businesses |

```typescript
  recommendedIncrease: number;
  newPrice: number;
  method: string;
  urgency: string;
  communicationTiming: string;
} {
  // Calculate inflation erosion since last increase
  const inflationErosion =
    (input.inflationRate / 12) * input.monthsSinceLastIncrease;

  const baseIncrease = Math.max(input.costIncreasePercent, inflationErosion);

  const newPrice = Math.round(
    input.currentPrice * (1 + recommendedIncrease / 100) * 100
  ) / 100;

  let urgency: string;

  if (input.monthsSinceLastIncrease > 18) {
    urgency = 'URGENTE: Han pasado mas de 18 meses sin ajuste.';
  } else if (input.monthsSinceLastIncrease > 12) {
```

Understanding typical cash flow patterns helps Kitz AI predict and prepare for lean periods.

**Restaurants / Food Service:**
- Relatively stable monthly patterns with predictable weekly cycles

- Weekly cycle: Thursday-Saturday are peak days (70% of weekly revenue)
- December is strongest month (holiday parties, events)
- January/February often weakest (post-holiday budget tightening)
- QuinceaÃ±eras and wedding season (Apr-Jun in Mexico) drive spikes

**Construction / Trades:**
- Highly project-dependent with lumpy cash flows

## 2.2 Working Capital Optimization

```typescript
// ============================================================
// Working Capital Metrics Calculator for Kitz AI
// ============================================================

interface FinancialData {
  // Income statement (monthly or period)
  revenue: number;
  costOfGoodsSold: number;
  creditSalesPercent: number; // What % of sales are on credit (vs. cash)

  // Balance sheet snapshots
  accountsReceivable: number;
  inventory: number;
  accountsPayable: number;

  // Period
  periodDays: number; // typically 30, 90, or 365
}

interface WorkingCapitalMetrics {
  dso: number;                 // Days Sales Outstanding
  dio: number;                 // Days Inventory Outstanding
  dpo: number;                 // Days Payable Outstanding
  ccc: number;                 // Cash Conversion Cycle
  workingCapital: number;      // Current working capital need
  workingCapitalRatio: number;
  recommendations: string[];
}

function calculateWorkingCapitalMetrics(
  data: FinancialData
): WorkingCapitalMetrics {
  const creditSales = data.revenue * (data.creditSalesPercent / 100);

  // DSO = Accounts Receivable / (Credit Sales / Period Days)
  const dso = creditSales > 0
    ? Math.round((data.accountsReceivable / (creditSales / data.periodDays)) * 10) / 10
    : 0;

  // DIO = Inventory / (COGS / Period Days)
  const dio = data.costOfGoodsSold > 0
    ? Math.round((data.inventory / (data.costOfGoodsSold / data.periodDays)) * 10) / 10
    : 0;

  // DPO = Accounts Payable / (COGS / Period Days)
  const dpo = data.costOfGoodsSold > 0
    ? Math.round((data.accountsPayable / (data.costOfGoodsSold / data.periodDays)) * 10) / 10
    : 0;

  // CCC = DSO + DIO - DPO
  const ccc = Math.round((dso + dio - dpo) * 10) / 10;

  // Working capital = AR + Inventory - AP
  const workingCapital =
    data.accountsReceivable + data.inventory - data.accountsPayable;

  const workingCapitalRatio =
    Math.round(
      ((data.accountsReceivable + data.inventory) / (data.accountsPayable || 1)) * 100
    ) / 100;

  // Generate recommendations
  const recommendations: string[] = [];

  if (dso > 45) {
    recommendations.push(
      `DSO de ${dso} dias es elevado. Meta: <45 dias. Implementar cobros por WhatsApp y descuentos por pronto pago.`
    );
  }
  if (dio > 30 && data.inventory > 0) {
    recommendations.push(
      `DIO de ${dio} dias indica exceso de inventario. Considerar modelo just-in-time o reducir SKUs de baja rotacion.`
    );
  }
  if (dpo < 30) {
    recommendations.push(
      `DPO de ${dpo} dias es bajo. Negociar plazos de pago mas largos con proveedores (meta: 30-45 dias).`
    );
  }
  if (ccc > 60) {
    recommendations.push(
      `Ciclo de conversion de efectivo de ${ccc} dias es alto. Cada dia que se reduce libera capital de trabajo.`
    );
  }
  if (workingCapitalRatio < 1.2) {
    recommendations.push(
      `Ratio de capital de trabajo de ${workingCapitalRatio} es ajustado. Meta minima: 1.5 para operacion saludable.`
    );
  }

  return {
    dso,
    dio,
    dpo,
    ccc,
    workingCapital,
    workingCapitalRatio,
    recommendations,
  };
}
```

```typescript
  } else {
    recommendation =
      'ACEPTABLE: Rotacion cerca del objetivo. Oportunidad de mejora con optimizacion de SKUs y ordenes mas frecuentes.';
  }

  return {
    currentTurns: Math.round(currentTurns * 10) / 10,
    targetInventoryValue: Math.round(targetInventoryValue),
    excessInventory: Math.round(excessInventory),

    safetyStockDays,
```

### Cash Conversion Cycle Optimization Strategies

**The Three Levers:**

1. **Reduce DSO (collect faster)**
   - Offer early payment discounts (2/10 Net 30)
   - Send invoices immediately upon delivery
   - Use WhatsApp reminders (see Collections section)
   - Accept digital payments (reduce check processing time)
   - Require deposits for large orders (30-50% upfront)

2. **Reduce DIO (sell inventory faster)**
   - Implement ABC analysis: focus resources on high-value A-items
   - Adopt just-in-time ordering where possible
   - Set automated reorder points based on sales velocity
   - Run clearance promotions for slow-moving stock
   - Negotiate smaller, more frequent deliveries with suppliers

3. **Increase DPO (pay suppliers later, ethically)**
   - Negotiate 45-60 day terms with key suppliers
   - Use supplier credit lines strategically
   - Pay on the due date, not before (unless there is a discount)
   - Consolidate purchases for better negotiation leverage
   - Time payments to coincide with your cash inflow peaks

```typescript
function modelCCCImprovement(
  current: { dso: number; dio: number; dpo: number },
  improvements: { dsoDelta: number; dioDelta: number; dpoDelta: number },
  dailyRevenue: number
): {
  currentCCC: number;
  improvedCCC: number;
  cashFreed: number;
  annualizedBenefit: number;
} {
  const currentCCC = current.dso + current.dio - current.dpo;
  const improvedCCC =
    (current.dso - improvements.dsoDelta) +
    (current.dio - improvements.dioDelta) -
    (current.dpo + improvements.dpoDelta);

  const daysFreed = currentCCC - improvedCCC;
  const cashFreed = daysFreed * dailyRevenue;

  return {
    currentCCC,
    improvedCCC,
    cashFreed: Math.round(cashFreed),
    annualizedBenefit: Math.round(cashFreed * 0.15), // Assume 15% cost of capital
  };
}

// Example: A retail store in Medellin, Colombia
const cccImprovement = modelCCCImprovement(
  { dso: 45, dio: 60, dpo: 30 },           // Current: CCC = 75 days
  { dsoDelta: 10, dioDelta: 15, dpoDelta: 10 }, // Improvements
  { dailyRevenue: 500_000 } as any,         // COP 500K/day
);
// Would free: (75 - 40) * 500,000 = COP 17,500,000 in working capital
```

### Early Payment Discounts

**2/10 Net 30 Explained:**
Offer a 2% discount if the customer pays within 10 days; otherwise full payment is due
in 30 days.

**Financial impact:** Offering 2% to accelerate payment by 20 days translates to an
annualized financing cost of approximately 36%. This is only worthwhile if:
- Your cost of capital exceeds 36% (common in LatAm with high interest rates)
- Cash flow timing is critical (e.g., you need to pay suppliers or make payroll)
- The customer has a history of very late payment

**LatAm variations:**

| Discount Structure | Typical Use | Effective Annual Rate |
|---|---|---|
| 2/10 Net 30 | Standard | ~36% |
| 3/10 Net 45 | B2B Colombia/Brazil | ~31% |
| 5/10 Net 60 | Large invoices, LatAm B2B | ~36% |
| 1/5 Net 30 | Small amounts, retain more margin | ~15% |
| 10% upfront deposit | Service businesses | Reduces risk, improves cash flow |

```typescript
function evaluateEarlyPaymentDiscount(
  invoiceAmount: number,
  discountPercent: number,
  discountDays: number,
  netDays: number,
  costOfCapitalAnnual: number // e.g., 0.25 for 25%
): {
  discountAmount: number;
  effectiveAnnualRate: number;
  isWorthOffering: boolean;
  recommendation: string;
} {
  const discountAmount = invoiceAmount * (discountPercent / 100);
  const daysSaved = netDays - discountDays;
  const effectiveAnnualRate =
    (discountPercent / (100 - discountPercent)) * (365 / daysSaved) * 100;

  const isWorthOffering = effectiveAnnualRate < costOfCapitalAnnual * 100;

  let recommendation: string;
  if (isWorthOffering) {
    recommendation =
      `SI ofrecer: El costo del descuento (${effectiveAnnualRate.toFixed(1)}% anualizado) ` +
      `es menor que su costo de capital (${(costOfCapitalAnnual * 100).toFixed(1)}%).`;
  } else {
    recommendation =
      `NO recomendado: El costo del descuento (${effectiveAnnualRate.toFixed(1)}% anualizado) ` +
      `excede su costo de capital (${(costOfCapitalAnnual * 100).toFixed(1)}%). ` +
      `Mejor usar una linea de credito.`;
  }

  return {
    discountAmount: Math.round(discountAmount * 100) / 100,
    effectiveAnnualRate: Math.round(effectiveAnnualRate * 10) / 10,
    isWorthOffering,
    recommendation,
  };
}
```

### Late Payment Penalties by Country

**Mexico:**
- Commercial Code Article 362: Default interest at the contractually agreed rate; in
  its absence, 6% per annum.
- Federal Civil Code Article 2395: Legal interest is 9% per annum.
- In practice, maximum moratory interest is approximately 2.5% monthly on the owed amount.
- Conventional interest can be higher if agreed between parties, but disproportionate
  rates can be challenged by the debtor.

**Brazil:**
- Civil Code Article 406: Legal default interest linked to SELIC rate (currently ~13.25%).
- Consumer relationships: maximum of 1% monthly moratory interest plus monetary correction.
- Commercial relationships: parties may agree on higher rates, but subject to judicial review.

**Colombia:**
- Maximum legal interest rate set by the Superintendencia Financiera, updated quarterly.
- Usury rate (interes de usura): typically 1.5x the banking current interest rate.
- Late payments may accrue interest at the maximum legal rate unless contractually lower.

**Argentina:**
- Due to hyperinflation, interest rates are extremely high and frequently adjusted.
- Commercial default interest often contractually linked to the Banco Nacion passive rate
  or similar benchmark.
- Courts may adjust excessive contractual interest rates.

**General LatAm best practice:** Always specify late payment interest in your contracts
and invoices. In the absence of explicit terms, legal default rates apply, which are
often lower than market rates.

## 2.4 Cash Flow Forecasting

### 13-Week Cash Flow Forecast

The 13-week (one quarter) rolling forecast is the gold standard for SMB cash management.
It provides enough granularity for weekly decision-making while covering a full business
quarter.

**Key principles:**
- Update weekly with actual data
- Always start from actual bank balance (not accounting balance)
- Forecast collections based on historical patterns, not invoice dates
- Include all cash movements, not just operating items
- Maintain three scenarios: pessimistic, likely, and optimistic

```typescript
// ============================================================
// 13-Week Cash Flow Forecast Engine for Kitz AI
// ============================================================

interface WeeklyForecast {
  weekNumber: number;
  weekStartDate: string;

  // Inflows
  cashSales: number;
  collectionsFromAR: number;
  otherInflows: number;
  totalInflows: number;

  // Outflows
  payroll: number;
  rent: number;
  supplierPayments: number;
  utilities: number;
  loanPayments: number;
  taxes: number;
  otherOutflows: number;
  totalOutflows: number;

  // Net
  netCashFlow: number;
  openingBalance: number;
  closingBalance: number;

  // Alerts
  alerts: string[];
}

interface ForecastScenario {
  name: 'pessimistic' | 'likely' | 'optimistic';
  revenueMultiplier: number;
  collectionDelayDays: number;
  costMultiplier: number;
}

const SCENARIOS: ForecastScenario[] = [
  {
    name: 'pessimistic',
    revenueMultiplier: 0.80,
    collectionDelayDays: 14,
    costMultiplier: 1.05,
  },
  {
    name: 'likely',
    revenueMultiplier: 1.0,
    collectionDelayDays: 0,
    costMultiplier: 1.0,
  },
  {
    name: 'optimistic',
    revenueMultiplier: 1.15,
    collectionDelayDays: -7, // Collect faster
    costMultiplier: 0.97,
  },
];

interface CashFlowForecastInput {
  currentBankBalance: number;
  weeklyBaseline: {
    expectedCashSales: number;
    expectedCollections: number;
    payroll: number;
    rent: number;           // Monthly, system divides by ~4.33
    supplierPayments: number;
    utilities: number;      // Monthly, system divides by ~4.33
    loanPayments: number;   // Monthly, system divides by ~4.33
    taxes: number;          // Monthly, system divides by ~4.33
    otherOutflows: number;
  };
  seasonalityFactors: number[]; // 13 multipliers, one per week (e.g., 1.0, 0.8, 1.3)
  knownOneTimeItems: { week: number; amount: number; description: string; isInflow: boolean }[];
  minimumCashReserve: number;
}

function generate13WeekForecast(
  input: CashFlowForecastInput,
  scenario: ForecastScenario = SCENARIOS[1]
): WeeklyForecast[] {
  const weeks: WeeklyForecast[] = [];
  let runningBalance = input.currentBankBalance;

  for (let w = 0; w < 13; w++) {
    const seasonality = input.seasonalityFactors[w] || 1.0;
    const revenueAdjust = scenario.revenueMultiplier * seasonality;
    const costAdjust = scenario.costMultiplier;

    const cashSales = Math.round(input.weeklyBaseline.expectedCashSales * revenueAdjust);
    const collections = Math.round(input.weeklyBaseline.expectedCollections * revenueAdjust);
    const otherInflows = input.knownOneTimeItems
      .filter((item) => item.week === w && item.isInflow)
      .reduce((sum, item) => sum + item.amount, 0);

    const totalInflows = cashSales + collections + otherInflows;

    const payroll = Math.round(input.weeklyBaseline.payroll * costAdjust);
    const rent = Math.round((input.weeklyBaseline.rent / 4.33) * costAdjust);
    const suppliers = Math.round(input.weeklyBaseline.supplierPayments * costAdjust * seasonality);
    const utilities = Math.round((input.weeklyBaseline.utilities / 4.33) * costAdjust);
    const loans = Math.round(input.weeklyBaseline.loanPayments / 4.33);
    const taxes = Math.round(input.weeklyBaseline.taxes / 4.33);
    const otherOut =
      Math.round(input.weeklyBaseline.otherOutflows * costAdjust) +
      input.knownOneTimeItems
        .filter((item) => item.week === w && !item.isInflow)
        .reduce((sum, item) => sum + item.amount, 0);

    const totalOutflows = payroll + rent + suppliers + utilities + loans + taxes + otherOut;
    const netCashFlow = totalInflows - totalOutflows;
    const closingBalance = runningBalance + netCashFlow;

    const alerts: string[] = [];
    if (closingBalance < input.minimumCashReserve) {
      alerts.push(
        `ALERTA: Saldo proyectado ($${closingBalance.toLocaleString()}) por debajo ` +
        `del minimo requerido ($${input.minimumCashReserve.toLocaleString()}) en semana ${w + 1}.`
      );
    }
    if (closingBalance < 0) {
      alerts.push(
        `CRITICO: Saldo negativo proyectado ($${closingBalance.toLocaleString()}) en semana ${w + 1}. ` +
        `Accion inmediata requerida.`
      );
    }
    if (netCashFlow < 0 && w > 0 && weeks[w - 1].netCashFlow < 0) {
      alerts.push(
        `ATENCION: Dos semanas consecutivas con flujo negativo. Revisar fuentes de ingreso.`
      );
    }

    const forecast: WeeklyForecast = {
      weekNumber: w + 1,
      weekStartDate: getWeekStartDate(w),
      cashSales,
      collectionsFromAR: collections,
      otherInflows,
      totalInflows,
      payroll,
      rent,
      supplierPayments: suppliers,
      utilities,
      loanPayments: loans,
      taxes,
      otherOutflows: otherOut,
      totalOutflows,
      netCashFlow,
      openingBalance: runningBalance,
      closingBalance,
      alerts,
    };

    weeks.push(forecast);
    runningBalance = closingBalance;
  }

  return weeks;
}

function getWeekStartDate(weeksFromNow: number): string {
  const date = new Date();
  date.setDate(date.getDate() + weeksFromNow * 7);
  return date.toISOString().split('T')[0];
}

// Utility: Run all three scenarios and compare
function runScenarioAnalysis(
  input: CashFlowForecastInput
): Record<string, { finalBalance: number; lowestBalance: number; weeksNegative: number }> {
  const results: Record<string, any> = {};

  for (const scenario of SCENARIOS) {
    const forecast = generate13WeekForecast(input, scenario);
    const finalBalance = forecast[12].closingBalance;
    const lowestBalance = Math.min(...forecast.map((w) => w.closingBalance));
    const weeksNegative = forecast.filter((w) => w.closingBalance < 0).length;

    results[scenario.name] = { finalBalance, lowestBalance, weeksNegative };
  }

  return results;
}
```

### Factoring / Invoice Discounting in LatAm

Invoice factoring allows businesses to sell their accounts receivable to a third party
(factor) at a discount in exchange for immediate cash.

**LatAm factoring market:** Projected to exceed USD $60 billion by 2030. MSMEs make up
99.5% of businesses in LatAm and are the primary beneficiaries.

**Key platforms and providers by country:**

| Country | Key Players | Typical Advance Rate | Typical Fee |
|---|---|---|---|
| Mexico | E Factor Network, Konfio, Credijusto | 80-90% | 1.5-4% per month |
| Brazil | Nexoos, BizCapital, QI Tech | 75-90% | 1.5-5% per month |
| Colombia | Mesfix, Portal Finance, Sempli | 80-90% | 2-4% per month |
| Chile | Buda.com, Cumplo, Banca Me | 80-90% | 1.5-3% per month |
| Peru | Facturedo, Innova Factoring | 75-85% | 2-5% per month |
| Pan-LatAm | Mercado Libre (Mercado Credito) | Varies | Integrated into platform |

**Regulatory landscape:**
- Mexico: Fintech Law (2018) regulates financial technology institutions
- Brazil: Resolution No. 4,991 (2022) allows fintechs to offer receivables financing
- Colombia: Decree 1154 (2020) regulates electronic trading of invoices; DIAN's RADIAN

**When to use factoring:**
- You have creditworthy customers but they pay slowly (Net 60+)
- You need cash to fund growth or seasonal inventory purchases
- Your cost of factoring is less than the cost of missing opportunities
- You cannot access or do not qualify for traditional bank credit

```typescript
interface FactoringAnalysis {
  invoiceAmount: number;
  daysUntilPayment: number;
  factoringFeePercent: number; // monthly
  advanceRatePercent: number;
  alternativeCostOfCapital: number; // annual %
}

function evaluateFactoring(input: FactoringAnalysis): {
  advanceAmount: number;
  factoringCost: number;
  effectiveAnnualRate: number;
  netReceived: number;
  recommendation: string;
} {
  const advanceAmount = input.invoiceAmount * (input.advanceRatePercent / 100);
  const months = input.daysUntilPayment / 30;
  const factoringCost = advanceAmount * (input.factoringFeePercent / 100) * months;
  const netReceived = advanceAmount - factoringCost;
  const effectiveAnnualRate = (input.factoringFeePercent * 12);

  let recommendation: string;
  if (effectiveAnnualRate < input.alternativeCostOfCapital) {
    recommendation =
      `Factoring RECOMENDADO: Tasa efectiva (${effectiveAnnualRate.toFixed(1)}%) ` +
      `es menor que su costo alternativo de capital (${input.alternativeCostOfCapital}%).`;
  } else {
    recommendation =
      `Factoring NO recomendado como primera opcion: Tasa efectiva (${effectiveAnnualRate.toFixed(1)}%) ` +
      `excede su costo alternativo. Considerar solo si no tiene otras opciones de financiamiento.`;
  }

  return {
    advanceAmount: Math.round(advanceAmount),
    factoringCost: Math.round(factoringCost),
    effectiveAnnualRate: Math.round(effectiveAnnualRate * 10) / 10,
    netReceived: Math.round(netReceived),
    recommendation,
  };
}
```

### Cost-Cutting Hierarchy

When cash is tight, cut costs in this order (least impactful to operations first):

```
Priority 1 - IMMEDIATE (No operational impact)
  - Cancel unused subscriptions and software
  - Renegotiate insurance, internet, phone plans
  - Eliminate non-essential travel and entertainment
  - Pause marketing experiments (keep proven channels)

Priority 2 - SHORT-TERM (Minor operational impact)
  - Reduce inventory orders to minimum viable levels
  - Defer non-essential maintenance (not safety-related)
  - Negotiate payment deferrals with flexible suppliers
  - Switch from monthly to per-use services where possible
  - Sublease unused office/warehouse space

Priority 3 - MEDIUM-TERM (Moderate operational impact)
  - Reduce working hours for non-essential staff
  - Renegotiate rent (LatAm landlords often prefer reduced rent over vacancy)
  - Consolidate supplier relationships for volume discounts
  - Convert fixed costs to variable where possible

Priority 4 - LAST RESORT (Significant operational impact)
  - Reduce headcount (furlough before layoff where legally possible)
  - Close underperforming locations or service lines
  - Sell non-essential equipment or assets
  - Seek debt restructuring with lenders

NEVER CUT:
  - Safety-related expenses
  - Legal/tax compliance costs
  - Core customer-facing quality
  - Key employee retention measures
```

### Emergency Cash Flow Decision Framework

```typescript
interface CashFlowEmergency {
  currentCashBalance: number;
  weeklyBurnRate: number;      // Average weekly net cash outflow
  urgentPayablesDue: number;   // Payments due within 7 days
  collectableAR: number;       // AR that could be collected with effort
  factoringAvailable: boolean;
  creditLineAvailable: boolean;
  creditLineLimit: number;
}

function assessEmergencyStatus(input: CashFlowEmergency): {
  runwayWeeks: number;
  severityLevel: 'green' | 'yellow' | 'orange' | 'red' | 'critical';
  immediateActions: string[];
  financingOptions: string[];
} {
  const runwayWeeks = input.weeklyBurnRate > 0
    ? Math.floor(input.currentCashBalance / input.weeklyBurnRate)
    : 52; // If not burning cash, assume comfortable

  const canPayUrgent = input.currentCashBalance >= input.urgentPayablesDue;

  let severityLevel: 'green' | 'yellow' | 'orange' | 'red' | 'critical';
  if (runwayWeeks >= 12 && canPayUrgent) severityLevel = 'green';
  else if (runwayWeeks >= 6 && canPayUrgent) severityLevel = 'yellow';
  else if (runwayWeeks >= 3) severityLevel = 'orange';
  else if (runwayWeeks >= 1) severityLevel = 'red';
  else severityLevel = 'critical';

  const immediateActions: string[] = [];
  const financingOptions: string[] = [];

  if (severityLevel === 'critical' || severityLevel === 'red') {
    immediateActions.push('URGENTE: Cobrar todas las cuentas vencidas hoy. Llamar a cada cliente.');
    immediateActions.push('Pausar TODOS los gastos no esenciales inmediatamente.');
    immediateActions.push('Negociar con proveedores para diferir pagos 15-30 dias.');
    immediateActions.push('Considerar venta relampago para generar efectivo inmediato.');

    if (input.factoringAvailable) {
      financingOptions.push(
        `Factoring disponible: Puede adelantar hasta $${Math.round(input.collectableAR * 0.85).toLocaleString()} de sus cuentas por cobrar.`
      );
    }
    if (input.creditLineAvailable) {
      financingOptions.push(
        `Linea de credito disponible: $${input.creditLineLimit.toLocaleString()}. Usar solo lo necesario.`
      );
    }
  } else if (severityLevel === 'orange') {
    immediateActions.push('Acelerar cobros: enviar recordatorios WhatsApp a todos los clientes con saldo vencido.');
    immediateActions.push('Revisar gastos y eliminar no esenciales.');
    immediateActions.push('Preparar plan de contingencia si la situacion empeora.');
  } else if (severityLevel === 'yellow') {
    immediateActions.push('Monitorear flujo semanalmente.');
    immediateActions.push('Optimizar ciclo de cobros (reducir DSO).');
    immediateActions.push('Construir reserva de emergencia (meta: 3 meses de gastos).');
  }

  return {
    runwayWeeks,
    severityLevel,
    immediateActions,
    financingOptions,
  };
}
```
